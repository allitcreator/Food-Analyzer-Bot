# 🥗 Food Analyzer Bot

Telegram-бот для отслеживания питания с ИИ-анализом еды, расчётом КБЖУ и персональными целями.

## Возможности

### 🤖 Распознавание еды
- **Текст** — напишите что съели, ИИ рассчитает КБЖУ
- **Фото** — сфотографируйте блюдо или упаковку, ИИ распознаёт визуально
- **Голос** — надиктуйте голосовое сообщение (Gemini 2.0 Flash)
- **Штрихкод** — сфотографируйте штрихкод → поиск в Open Food Facts → GPT-анализ
- **Несколько блюд** — одно сообщение может содержать несколько позиций, каждую можно отредактировать отдельно

### 📊 Статистика и отчёты
- `/stats` — дневная статистика с прогресс-барами и серией дней 🔥
- `/week` — разбивка по дням за 7 дней
- `/month` — статистика за месяц с 4-недельной разбивкой
- `/pdf` — PDF-отчёт с графиками (калории, КБЖУ, динамика веса, корреляция)
- `/export` — Excel-экспорт за дату или диапазон

### 🔬 Микронутриенты
- Клетчатка, сахар, натрий, насыщенные жиры — оцениваются ИИ для каждого продукта
- Включаются/выключаются через `/settings` (по умолчанию выключены)
- Отображаются в карточке распознавания, `/stats` и прогрессе дня
- Пропорционально пересчитываются при изменении веса порции

### ⚖️ Отслеживание веса
- `/weight [кг]` — записать вес или просмотреть историю с трендом
- ИИ-анализ динамики веса за неделю
- В PDF: корреляция веса и калорий, таблица по неделям

### 🍽 Оценка качества питания
- Каждому продукту ставится оценка 1–10
- Персональный совет по питанию на русском языке

### 👤 Профиль и цели
- `/profile` — полная настройка профиля (возраст, пол, вес, рост, активность, цель)
- `/editprofile` — редактирование отдельных полей
- Расчёт нормы калорий по формуле Миффлина-Сан Жеора
- `/goal` — быстрая смена цели (похудение / поддержание / набор)

### 🤖 ИИ-тренер
- `/ask [вопрос]` — вопрос нутрициологу с контекстом профиля и еды за день
- `/report` — вечерний ИИ-анализ питания (или автоматически по расписанию)

### ⏰ Напоминания
- `/reminders` — напоминания о завтраке, обеде, ужине
- Напоминание если не было записей к заданному времени
- `/weightreminder` — напоминание взвеситься (время + дни недели)
- `/report_time` — время автоматического вечернего отчёта

### 🏋️ Тренировки
- Распознавание из свободного текста или голоса: *"пробежал 5 км"*, *"30 мин на эллипсе"*, *"10000 шагов"*, *"потратил 600 ккал"*
- ИИ определяет тип активности и считает калории по MET-значениям с учётом веса пользователя
- Подтверждение перед сохранением — как при добавлении еды
- `/workout` — тренировки сегодня + чистые калории (съедено − сожжено)
- В `/stats` и прогрессе дня: блок тренировок и строка чистых калорий

### ⚙️ Настройки
- `/settings` — управление отображением микронутриентов

### 🔧 Администрирование
- Whitelist-система: пользователи запрашивают доступ, админ одобряет
- `/users` — управление пользователями

## Стек технологий

| Компонент | Технология |
|-----------|------------|
| Bot | node-telegram-bot-api |
| ИИ (текст/фото) | OpenRouter → openai/gpt-4o, openai/gpt-4o-mini |
| ИИ (голос) | OpenRouter → google/gemini-2.0-flash |
| База данных | PostgreSQL + Drizzle ORM |
| PDF | pdfkit + dejavu-fonts-ttf |
| Excel | exceljs |
| Штрихкоды | Open Food Facts API |
| Backend | Express + TypeScript |
| Frontend | React + Vite + Tailwind + shadcn/ui |

## Переменные окружения

```env
TELEGRAM_BOT_TOKEN=      # Токен от @BotFather (обязательно)
OPENROUTER_API_KEY=      # Ключ OpenRouter — все AI запросы (обязательно)
DATABASE_URL=            # PostgreSQL строка подключения (обязательно)
SESSION_SECRET=          # Секрет сессий Express (обязательно)
WEBHOOK_URL=             # HTTPS URL сервера для webhook (напр. https://example.com)
WEBHOOK_SECRET=          # Секрет webhook-эндпоинта
ADMIN_TELEGRAM_ID=       # Telegram ID администратора (опционально)
TZ=Europe/Moscow         # Таймзона для напоминаний
```

## Запуск через Docker (рекомендуется)

```bash
cp .env.example .env
nano .env          # заполнить токены и секреты
docker compose up -d --build
```

Подробная инструкция по деплою на VPS — см. [DEPLOY.md](DEPLOY.md).

## Локальная разработка

```bash
npm install
npm run db:push   # применить схему к базе
npm run dev
```
