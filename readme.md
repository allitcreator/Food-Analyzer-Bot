# Calorie Tracker Bot

Telegram-бот для трекинга питания с AI-распознаванием еды (текст, фото и голос), расчётом КБЖУ, персональными нормами и вечерними AI-отчётами.

## Возможности

- **AI-анализ еды**: GPT-4o для фото, GPT-4o-mini для текста и отчётов
- **Голосовые сообщения**: Whisper-1 транскрибирует аудио → анализируется как текст
- **Мультипозиционное распознавание**: из одного сообщения выделяется несколько блюд, каждое можно редактировать, корректировать вес и удалять перед сохранением
- **Оценка качества**: каждый продукт получает оценку 1–10 с персональным советом на русском
- **Профиль**: пол, возраст, вес, рост, уровень активности, цель — запускается автоматически после `/start`, если не заполнен
- **Расчёт норм КБЖУ**: формула Миффлина–Сан Жеора
- **Подтверждение порции**: inline-кнопки для корректировки веса/объёма перед сохранением
- **Умные единицы**: жидкости в мл, твёрдая пища в граммах
- **Напоминания о приёмах пищи**: настраиваемые напоминания на завтрак/обед/ужин
- **Вечерний отчёт**: AI-анализ рациона с рекомендациями; автоматически пропускается если нет записей, `/report` сообщает об отсутствии данных
- **Экспорт в Excel**: детальная выгрузка за выбранный период
- **Система доступа**: одобрение пользователей администратором

## Архитектура

### Backend (Express + Node.js)

- `server/index.ts` — HTTP-сервер, маршруты, статика
- `server/bot.ts` — логика бота: команды, профиль, мультипозиционное распознавание, редактирование позиций, напоминания, отчёты
- `server/openai.ts` — GPT-4o (фото), GPT-4o-mini (текст, отчёты), Whisper-1 (голос)
- `server/storage.ts` — CRUD, расчёт КБЖУ по Миффлину–Сан Жеору
- `server/db.ts` — PostgreSQL через Drizzle ORM

### Frontend (React + Vite)

Минимальный фронтенд — страница статуса бота. React, Vite, Tailwind CSS, shadcn/ui.

### Shared

- `shared/schema.ts` — Drizzle-схемы таблиц `users`, `foodLogs` + Zod-схемы

## База данных (PostgreSQL)

**users** — профиль (пол, возраст, вес, рост, активность, цель), рассчитанные нормы КБЖУ, настройки напоминаний и отчётов

**food_logs** — название, калории, БЖУ, вес (г/мл), тип приёма пищи, оценка качества (1–10), совет

## Настройка на Replit

1. Создать бота через @BotFather → получить `TELEGRAM_BOT_TOKEN`
2. Добавить `TELEGRAM_BOT_TOKEN` в секреты проекта
3. Добавить `OPENAI_API_KEY` в секреты (нужен для Whisper; GPT-модели работают через Replit AI Integration без личного ключа)
4. `ADMIN_TELEGRAM_ID` — Telegram ID для управления пользователями (опционально)
5. PostgreSQL подключён автоматически через Replit

## Важные детали реализации

- **GPT-4o vs GPT-4o-mini** — фото через дорогую модель с vision, текст/отчёты через дешёвую
- **Whisper** — работает только через прямой `OPENAI_API_KEY` (Replit-прокси не поддерживает audio API)
- **Два OpenAI-клиента** — `openai` (Replit-прокси для GPT) и `whisperClient` (прямой для аудио)
- **Мультипозиционный flow** — `pendingMulti[telegramId]` хранит массив `FoodItem[]`; кнопки `mi_e_N`, `mi_wp/wm_AMOUNT_N`, `mi_del_N`, `mi_back` управляют редактированием
- **Mifflin-St Jeor** — расчёт базового метаболизма в `storage.ts → calculateAndSetGoals()`
- **LIQUID_PATTERN** — regexp в `bot.ts` определяет мл/г для правильных единиц измерения
- **Вечерний планировщик** — проверка каждую минуту по московскому времени (UTC+3)
